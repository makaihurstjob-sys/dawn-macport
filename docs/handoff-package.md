## Fresh Supabase Setup

The old Supabase project is not accessible. The new project is:

- Project ref: `ayjsgmdpzlsokfgyjhsl`
- Project URL: `https://ayjsgmdpzlsokfgyjhsl.supabase.co`

1. Create a new Supabase project in the account that will own the website.
2. In Supabase SQL Editor, run:
   - `supabase/migrations/20260512235202_ab38c8ef-2538-46ef-a4f3-7feaabab0472.sql`
   - `supabase/migrations/20260512235236_add1a852-aab7-4e62-a39d-4c2d7f667605.sql`
   - `supabase/migrations/20260523053000_booking_name_settings_delete.sql`
3. In Supabase Auth, create the admin user.
4. In Supabase Auth, create the developer user.
5. In SQL Editor, assign roles:

```sql
insert into public.profiles (id, username, role)
select id, email, 'admin'::public.app_role
from auth.users
where email = 'OWNER_EMAIL_HERE'
on conflict (id) do update
set role = 'admin'::public.app_role,
    username = excluded.username;

insert into public.profiles (id, username, role)
select id, email, 'developer'::public.app_role
from auth.users
where email = 'DEVELOPER_EMAIL_HERE'
on conflict (id) do update
set role = 'developer'::public.app_role,
    username = excluded.username;
```

6. Copy these values into local `.env` and into the hosting provider environment variables:
   - `SUPABASE_URL`
   - `SUPABASE_PUBLISHABLE_KEY`
   - `SUPABASE_SERVICE_ROLE_KEY`
   - `VITE_SUPABASE_URL`
   - `VITE_SUPABASE_PUBLISHABLE_KEY`
   - `VITE_SUPABASE_PROJECT_ID`

Do not send the service role key to the client in a visible package. It is server-only.

## Netlify / Hosting Setup

This project is a TanStack Start app, not a plain static HTML site. The current local build is configured through Lovable's TanStack setup and Cloudflare-oriented output.

Before treating Netlify as final hosting, connect the project through Netlify's TanStack Start deployment path or install Netlify's TanStack Start Vite plugin in the production repo. Reference: https://docs.netlify.com/frameworks/tanstack-start/

See `docs/netlify-deploy-checklist.md` for the exact deployment checklist.

The required production environment variables are the same values listed above.

Suggested Netlify settings after the adapter is added:

- Build command: `npm run build`
- Publish/output directory: generated by the Netlify TanStack Start adapter
- Environment variables:
  - `VITE_SUPABASE_URL`
  - `VITE_SUPABASE_PUBLISHABLE_KEY`
  - `VITE_SUPABASE_PROJECT_ID`
  - `SUPABASE_URL`
